///    
///    // Method 3: Safe access
///    final service = ServiceLocator.getSafe<MyService>(
///      defaultValue: MyService(),
///    );
///    ```

/// 
/// SERVICE REGISTRATION PATTERNS:
/// 
/// SINGLETON (@singleton)
/// - Single instance for entire app lifecycle
/// - Used for: Repositories, Database, API clients
/// 
/// LAZY SINGLETON (@lazySingleton)
/// - Single instance, created on first access
/// - Used for: Heavy services initialized on demand
/// 
/// FACTORY (@factoryMethod)
/// - New instance each time it's requested
/// - Used for: Stateful providers, temporary objects
/// 
/// NAMED (@singleton or @lazySingleton with name)
/// - Multiple instances with different names
/// - Used for: Multiple database connections, different API versions
/// 

/// 
/// EXPECTED DIRECTORY STRUCTURE AFTER SETUP:

/// BEST PRACTICES:
/// 
/// ✓ Use abstract classes for repositories and data sources
/// ✓ Register as interfaces (@Singleton(as: Interface))
/// ✓ Use @Injectable() for use cases and smaller services
/// ✓ Initialize Firebase before configureInjection()
/// ✓ Use ServiceLocator.getSafe() for optional dependencies
/// ✓ Keep modules organized by feature/layer
/// ✓ Document why a singleton is needed
/// ✗ Don't mix @singleton and @lazySingleton carelessly
/// ✗ Don't create circular dependencies
